СПЕЦИФИКАЦИЯ ФОРМАТА ДАННЫХ ЛАНДШАФТА (.CDATA)
Движок: Gamma Engine
Версия формата: VERSION_ABS_QFLOAT (QPNG) + BwPackedSection
================================================================================

1. СТРУКТУРА ФАЙЛА
--------------------------------------------------------------------------------
Файл .cdata — это контейнер, содержащий разнородные данные (высоты, текстуры,
нормали). Данные могут храниться в "сыром" виде или быть упакованы в 
специальные секции (Packed Sections).

Основные секции:
1. "terrain2/heights" - Карта высот (Геометрия).
2. "terrain2/normals" - Карта нормалей (Освещение).
3. "terrain2/layer"   - Описание слоя текстурирования.

2. BW PACKED SECTION
--------------------------------------------------------------------------------
Многие данные (особенно в новых версиях формата) обернуты в структуру 
BwPackedSection.
Magic Number: 0x62a14e45 ("EN¡b")
Особенности:
- Содержит словарь строк.
- Поддерживает вложенность (дерево данных).
- Может содержать зашифрованные данные (Simple XOR key: 0x9c).

Движок сначала проверяет наличие заголовка PackedSection. Если он найден,
данные распаковываются и декодируются перед использованием.

3. КАРТА ВЫСОТ (HEIGHTMAP)
--------------------------------------------------------------------------------
Формат: QPNG (Quantized PNG).
Данные высот упакованы в стандартный PNG поток внутри секции.
Каждые 4 байта пикселя (RGBA) интерпретируются как одно число int32.
Масштаб: 1 единица = 1 миллиметр (0.001 метра).

4. СЛОИ (LAYERS)
--------------------------------------------------------------------------------
Каждый слой описывается блоком данных с заголовком "bld" (BlendHeader).
Движок ищет секции, содержащие "terrain2/layer".
Внутри блока находится:
- Имя текстуры (строка).
- Данные маски смешивания (PNG или Raw массив байт).
- Матрицы проекции текстуры.
