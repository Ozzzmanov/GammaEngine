================================================================================
СПЕЦИФИКАЦИЯ СИСТЕМЫ ВОДЫ (WATER SYSTEM)
Тип объекта: Very Large Object (VLO)
================================================================================

1. ФАЙЛОВАЯ СТРУКТУРА
--------------------------------------------------------------------------------
Водный объект описывается парой файлов с уникальным UID:

A. Файл настроек (*.vlo):
   - Формат: XML (читается через DataSection).
   - Содержит: позицию, размер, ссылки на текстуры (волны, отражения), 
     параметры шейдера (цвет глубины, скорость ветра, сила солнца).

B. Файл геометрии (*.odata):
   - Формат: BigWorld Binary (Magic: 0x42a14e65).
   - Секция "alpha": Битовая маска или массив байт, определяющий 
     прозрачность воды (альфа-канал) для создания плавного берега.

2. РЕНДЕРИНГ И ШЕЙДЕРЫ
--------------------------------------------------------------------------------
Вода рендерится отдельным проходом (Water Pass) после отрисовки ландшафта.
Ключевые технологии шейдера (Water.hlsl):

- Normal Map Scrolling: Анимация волн достигается смещением UV координат 
  двух карт нормалей навстречу друг другу.
- Fresnel Effect: Изменение прозрачности и отражающей способности в 
  зависимости от угла взгляда (вода прозрачная под ногами, зеркальная вдали).
- Environment Mapping: Имитация отражений неба через Cubemap.

3. ОПТИМИЗАЦИЯ (VISIBILITY CULLING)
--------------------------------------------------------------------------------
Водные объекты интегрированы в общую систему отсечения движка Gamma Engine.
Для каждого объекта вычисляется BoundingBox (AABB):
- Ширина/Длина: соответствуют размеру водной глади.
- Высота: имеет буфер (+/- 10 метров) для учета амплитуды волн.

Условия отмены рендера:
1. Frustum Culling: AABB воды полностью находится вне конуса камеры.
2. Distance Culling: Ближайшая точка AABB находится дальше глобальной 
   дистанции прорисовки (`m_renderDistance`).
