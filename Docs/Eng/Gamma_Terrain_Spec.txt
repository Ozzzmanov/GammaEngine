TERRAIN DATA FORMAT SPECIFICATION (.CDATA)
Engine: Gamma Engine (v0.3)
Format Version: VERSION_ABS_QFLOAT (QPNG) + BwPackedSection
================================================================================

1. FILE STRUCTURE
--------------------------------------------------------------------------------
The .cdata file is a container for heterogeneous data (heights, textures,
normals). Data can be stored in "raw" format or encapsulated in 
Packed Sections.

Key Sections:
1. "terrain2/heights" - Heightmap (Geometry).
2. "terrain2/normals" - Normal map (Lighting).
3. "terrain2/layer"   - Texture layer definition.

2. BW PACKED SECTION
--------------------------------------------------------------------------------
Many data blocks (especially in newer format versions) are wrapped in the 
BwPackedSection structure.
Magic Number: 0x62a14e45 ("EN¡b")
Features:
- Contains an internal string dictionary for tag compression.
- Supports recursive nesting (data tree).
- Uses safe parsing with memory bounds protection.

3. HEIGHTMAP
--------------------------------------------------------------------------------
Format: QPNG (Quantized PNG).
Height data is packed into a standard PNG stream within the section.
Each 4-byte pixel (RGBA) is reinterpreted as a single int32 value.
Scale: 1 unit = 1 millimeter (0.001 meters).
Version 0.3 uses the optimized `stb_image` library for decoding.

4. LAYERS
--------------------------------------------------------------------------------
Each layer is defined by a data block with a "bld" header (BlendHeader).
The engine scans for all "terrain2/layer" sections in the file.
Block contents:
- Texture filename (string).
- Blend mask data (PNG or Raw byte array, supports ZLib compression).
- Texture projection matrices (UProjection, VProjection).
