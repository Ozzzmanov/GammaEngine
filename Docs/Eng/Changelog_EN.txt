v0.4.0 ‚ÄĒ TRANSITION TO GPU-DRIVEN & ASYNCHRONY
--------------------------------------------------------------------------------
Fundamental overhaul of the rendering core and resource system.

1. ENGINE CONFIGURATION SYSTEM:
   - Added 'EngineConfig': full support for external settings via JSON ('engine.json').
   - Dynamic graphics management: FOV, Draw Distance, VSync, Anisotropy.
   - Keybindings moved to config (completely eliminating hardcoded keys in the source).

2. ASYNCHRONY & MULTITHREADING:
   - Implemented 'TaskScheduler': custom thread pool for background tasks.
   - Asynchronous texture loading: engine no longer freezes when streaming assets.
   - Thread-safe 'ResourceManager' utilizing 'shared_mutex' (RW-lock).

3. GPU-DRIVEN RENDERING PIPELINE:
   - Created 'RenderPipeline': the central hub managing all render passes.
   - Switched to Indirect Drawing: utilizing 'DrawIndexedInstancedIndirect'.
   - GPU Culling: Object culling (Frustum + Occlusion) moved to Compute Shaders.
   - Implemented HZB (Hi-Z Buffer) system for efficient Occlusion Culling.

4. TERRAIN MODERNIZATION:
   - Added 'TerrainBaker': an offline converter that normalizes .cdata into optimized DDS caches.
   - 'TerrainGpuScene': renders the entire map in a single Draw Call via Texture2DArrays.
   - Dynamic height and normal reading directly in the shader from texture arrays.

5. MODEL SYSTEM (STATIC MESH):
   - Implemented support for GLTF and GLB formats via 'tiny_gltf'.
   - Added 'ModelManager' for global geometry caching.
   - Support for up to 3 LOD levels for every static entity.
   - Instancing on steroids: thousands of objects rendered with minimal CPU overhead.

6. GRAPHICS EFFECTS & OPTIMIZATION:
   - Z-Prepass: initial depth pass to minimize Overdraw.
   - Deferred Lighting: lighting calculated in a separate pass based on the G-buffer.
   - Improved Samplers: support for 16x Anisotropy and MipLODBias for ultra-sharp textures.
   - Timer Update: strict delta clamping to prevent "falling through the floor" during lag spikes.

v0.3.1 ‚ÄĒ ARCHITECTURAL REFACTORING (Legacy)
- Moved Camera to a dedicated class.
- Implemented Action Mapping.
--------------------------------------------------------------------------------
Global code cleanup and Separation of Concerns.
Goal: Offload logic from the main Engine class.

1. INPUT SYSTEM:
   - Implemented Action Mapping (binding actions to keys).
   - Full removal of hardcoded keys (VK_F1, 'W', etc.) from game logic.
   - WinAPI abstraction: convenient IsActionActive() and IsActionTriggered() methods.

2. CAMERA SYSTEM MODULARITY:
   - Movement logic, View/Projection matrix calculation, and Frustum Culling moved
     from the engine core to a standalone 'Camera' class.
   - Isolation of math and debug mode management (Debug Camera).

3. WORLD LOADER:
   - Created 'WorldLoader' class responsible for directory scanning,
     chunk coordinate parsing, and VLO (Very Large Object) discovery.
   - GammaEngine now acts as an orchestrator, delegating loading to the subsystem.

4. USER INTERFACE (GUI):
   - Debug information rendering (Debug Overlay) moved to a dedicated 
     class within the new 'GUI/' directory.

v0.3 - OPTIMIZATION & WATER UPDATE
--------------------------------------------------------------------------------
Key Changes:

1. RENDERING OPTIMIZATION (VISIBILITY CULLING):
   - Implemented Frustum Culling (discarding objects outside the FOV).
   - Implemented Distance Culling (hard cutoff of objects based on range).
   - Performance boost: 5-10x increase in large-scale locations.
   - Added dynamic controls: Draw Distance (Z/X keys) and Optimization Toggle (F1).

2. WATER SYSTEM:
   - Implemented full support for VLO (Very Large Objects) of "Water" type.
   - Parsing: Reading parameters from '.vlo' (XML) and geometry from '.odata' (Binary).
   - Water Shader: Normal maps, scrolling animation, Environment Map reflections, 
     and Fresnel effect.
   - Integration: Water respects the culling system (not rendered if not visible).

3. CORE & ARCHITECTURE:
   - Modularity: Separated entry point (Main.cpp) from the engine class (GammaEngine).
   - Async Logger: Thread-safe logging to file and debug window 
     (prevents UI freezes during text output).
   - Safe Parsing: 'DataSection' correctly handles floating-point numbers 
     regardless of Windows system locale (fixed comma/period decimal issue).
   - Timer: Added delta smoothing for fluid camera movement and physics.

4. RESOURCES:
   - Rewrote BwBinSection parser: Optimized memory safety and speed.
   - Optimized HeightMap: Fast decoding of QPNG heightmaps.
   - Dynamic Buffers: ConstantBuffers moved to DYNAMIC mode (Map/Unmap) 
     for efficient per-frame updates.

================================================================================
ARCHITECTURAL DIFFERENCES FROM ORIGINAL TECHNOLOGY (BigWorld)
================================================================================

1. GRAPHICS API (RENDERING BACKEND)
--------------------------------------------------------------------------------
[ORIGINAL]
Uses DirectX 9.0c.
- Fixed-function pipeline or legacy shaders (SM 2.0/3.0).
- Lack of hardware-level support for modern GPU features.
- Dependency on deprecated D3DX9 libraries.

[GAMMA ENGINE]
Uses DirectX 11.
- Fully Programmable Pipeline.
- Use of Shader Model 5.0 (HLSL).
- Support for modern buffer formats and state objects.

2. TERRAIN TEXTURING SYSTEM
--------------------------------------------------------------------------------
[ORIGINAL]
Multi-pass rendering or Texture Atlases.
- DX9 Limitation: Inability to bind many textures to a single shader.
- Rendering: Terrain was drawn in multiple passes (4 layers each) or 
  used complex atlases.
- Performance drops with a high number of layers.

[GAMMA ENGINE]
Texture Arrays.
- All chunk textures are loaded into a single resource array.
- Rendering is performed in a SINGLE pass (Single Draw Call) regardless of 
  the number of layers (up to 8 and more).
- Shader selects the layer by index, significantly reducing CPU overhead.

3. LAYER BLENDING
--------------------------------------------------------------------------------
[ORIGINAL]
Using a single mask (RGBA) limited the number of layers to 4.
A 5th layer required an additional geometry pass (Overdraw).

[GAMMA ENGINE]
Dual Blend Maps system.
- Two blend textures are used simultaneously.
- Allows blending up to 8 layers in a single pixel without re-drawing geometry.

4. MATHEMATICAL FOUNDATION
--------------------------------------------------------------------------------
[ORIGINAL]
D3DX Math library.
- Legacy functions executed on the FPU. Slow vector operations.

[GAMMA ENGINE]
DirectXMath (SIMD).
- Utilizes SSE2/AVX CPU instructions.
- High performance for physics and transformations.

5. FILE & DATA HANDLING
--------------------------------------------------------------------------------
[ORIGINAL]
Complex dependency on Python and proprietary parsers.
- Hardcoded binding to 'res/' folder structures.
- Legacy archiving mechanisms.

[GAMMA ENGINE]
Native Modern C++ (C++17/20).
- Direct parsing of binary formats (.cdata, .chunk) via custom classes.
- Use of std::filesystem for cross-platform compatibility.
- On-the-fly decoding of compressed heightmaps (QPNG) without conversion.

6. MEMORY MANAGEMENT
--------------------------------------------------------------------------------
[ORIGINAL]
Raw pointers and manual reference counting, often leading to memory leaks.

[GAMMA ENGINE]
RAII and Smart Pointers.
- Widespread use of std::unique_ptr and ComPtr.
- Automatic resource deallocation.
